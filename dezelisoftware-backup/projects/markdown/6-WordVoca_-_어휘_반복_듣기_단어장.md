# WordVoca - 영어 단어장

**기간**: 2021-01-01 ~ 2021-06-30

**기술 스택**: Python Tkinter, Sqlite3, gTTS

**GitHub**: [Dezeli/WordVoca](https://github.com/Dezeli/WordVoca)

### 영어 공부용으로 만든 듣는 단어장

내 영어 공부에 쓰려고 만든, 단어를 음성으로 반복 재생해 주는 단어장이다.

단어를 추가하면 gTTS로 mp3가 생성되고, 듣기 버튼을 누르면 영어와 한국어 뜻이 지정한 횟수만큼, 지정한 간격으로 반복 재생된다. 외우려는 단어 묶음을 그대로 흘려서 들을 수 있게 만들고 싶었다.

엑셀 대량 등록도 붙여서, 한 번에 수십 단어를 밀어 넣을 수 있게 했다.

### 기술 스택

기존 프로젝트의 Python + Tkinter + SQLite3 골격은 그대로 두고, 음성 처리와 엑셀 입력만 새로 얹은 구성이다.

- **Language**: Python
- **Library**: Tkinter, Pillow, Pyglet, gTTS, playsound, openpyxl, PyInstaller
- **Database**: SQLite3
- **Module**: threading, multiprocessing, os, time, datetime

### 카카오 TTS에서 gTTS로

처음에는 카카오 TTS API로 시작했다. 음질이 훨씬 자연스러웠고 단어장이라는 용도에 더 맞았다. API 호출까지 성공시키고 붙여 보던 단계까지 갔다.

그런데 결국 **gTTS로 회귀했다.** 학생 입장에서 호출당 비용이 붙는 API를 계속 끌고 가는 건 당연히 부담이었다. 음질을 한 단계 양보하는 대신 무료를 택했다.

### Tkinter에서 끄는 게 제일 어려웠다

기능 자체보다 까다로웠던 건 **재생 중간에 강제로 끄는 처리**였다.

Tkinter 안에서 thread로 음성 재생을 돌리면, 사용자가 정지 버튼을 눌러도 깔끔하게 멈추지 않았다. 재생 한 사이클이 끝나기까지 무시되거나, 끄긴 끄는데 다음 클릭이 안 먹거나 — 그런 어긋남이 계속 났다.

결국 thread 대신 **multiprocessing**으로 갈아탔다. 프로세스 단위로 띄워서 `terminate()`로 강제 종료하는 방식. 우아한 해법은 아니었지만, 누른 버튼이 의도대로 동작하는 단어장은 그제야 만들 수 있었다.

### 결국 본인은 잘 안 썼다

솔직히 말하면, 만들어 놓고 정작 본인은 잘 안 썼다.

써보긴 했는데 gTTS 음질의 한계가 단어장 용도에서는 생각보다 컸다. 발음을 귀에 익히려고 듣는 단어장인데, 그 발음이 충분히 또렷하지 않으니 자연스럽게 손이 안 갔다. 음질을 양보한 결정의 후유증을 제일 먼저 본인이 받은 셈이다.

### 그래도 남은 것

이 시기쯤부터, 새 프로젝트를 시작하는 게 막막하지 않게 느껴지기 시작했다. 코드도 이전보다 훨씬 정돈됐다 — 등록·재생·정렬·삭제·대량 입력을 하나의 GUI 안에 묶는데, 더 이상 처음 다뤄 보는 영역이 거의 없었다.
