어제, 백준 알고리즘이 운영을 종료했다.

처음 그 소식을 들었을 땐 안 믿었다. 워낙 큰 이슈라 여기저기서 자꾸 들리는데도, 한참은 농담 같았다. 같은 말이 하도 반복되니까 그제야 마음 한구석이 천천히 허전해지기 시작했다.

### 2. Codeup에서 백준으로

처음 백준에 발을 들인 건 고등학교 1학년 때였다. 그 전엔 중학생 내내 **Codeup**에서 살았는데, 어느 순간 그 안에서 풀 게 마땅찮아졌다. 자연스럽게 다음 문이 백준이었다.

파이썬을 막 배우던 무렵이었다. 한 문제 한 문제 AC가 떠오를 때마다 따라오던 그 작은 보상감에, 어느새 깊게 빠져들고 있었다.

### 3. 일주일을 갈아 넣었던 골드 문제

처음으로 야심차게 도전한 골드 문제가 **골롬 수열**이었다. 당시 내 실력에는 너무 컸다.

다 짠 줄 알고 제출하면 시간 초과, 자료구조를 바꿔서 내면 메모리 초과. 정확히 어디가 문제였는지 지금은 떠오르지도 않을 만큼, 제출과 빨간 글씨 사이를 일주일 넘게 오갔다. 자다가도 머릿속에 그 수열이 떠다녔다. 결국 어떻게든 풀어낸 코드는 지금도 남아 있다.

```python
# 골롬 수열
import sys

n = int(sys.stdin.readline().rstrip())


def golomb():
    suyeol = [(0, 0), (1, 1), (2, 3), (4, 5)]
    idx = 3
    last = 5
    while True:
        start, end = suyeol[idx]
        for _ in range(start, end + 1):
            if last + 1 <= n <= last + idx:
                return len(suyeol)
            suyeol.append((last + 1, last + idx))
            last += idx
        idx += 1


if n == 1:
    result = 1
elif n == 2 or n == 3:
    result = 2
else:
    result = golomb()
print(result)
```

문제는 이제 다시 펼쳐볼 수 없다. 그래도 이 코드는 내 안에 남았다.

### 4. 풀고 싶어서 미쳐 있었을 뿐

돌이켜보면 신기한 건, 그만두고 싶었던 적이 한 번도 없었다는 것이다. 그저 **풀고 싶어서 미쳐 있었을 뿐이었다.**

좋아하는 영역도 분명했다. 트리, 그중에서도 **세그먼트 트리**가 잘 맞았다. 구간 쿼리가 한 줄에 떨어지는 그 깔끔함이 마음에 들었다. 반대로 DP에는 끝까지 약했다. 점화식을 노트에 끄적거리다가 다음 날 다시 보면 무슨 정신으로 적었는지 모를 때가 많았다.

그렇게 한 문제 한 문제가 쌓여, 결국 플래티넘까지 나를 끌고 갔다.

### 5. 마지막 한 문제

그 시간들 끝에 **아주대학교 프로그래밍 경진대회 APC EXTRA**가 있었다. 약 60명이 참가한 개인전이었고, 결과는 대상이었다.

솔직히 1등을 노리고 간 것도 아니어서 더 얼떨떨했다. 더 웃긴 건 정작 그날 한 문제를 못 풀었다는 점이다. 코드 자체는 어렵지 않았다. 단 하나의 예외 케이스를 찾아 처리할 수 있느냐를 묻는 문제였다.

그런데 **'마지막 문제 = 최종 보스'** 라는 선입견에 갇혀, 일부러 어렵게 어렵게 풀려고 하다가 그 단순한 예외 하나를 끝내 보지 못했다.

지나고 보니, 이 한 문제가 백준이 나에게 가르친 걸 가장 잘 요약해 주는 것 같다. 코드보다 자주 발목을 잡는 건, 결국 머릿속의 *'어렵겠지'* 였다.
