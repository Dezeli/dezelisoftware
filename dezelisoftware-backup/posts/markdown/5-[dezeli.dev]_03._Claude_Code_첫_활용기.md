# [dezeli.dev] 03. Claude Code 첫 활용기

월 22달러. 싸다고는 할 수 없는 금액이다. 하지만 *"이것도 경험이다"* 라는 생각 하나로 Claude Code를 결제했다. 과연 이 돈값을 할 수 있을까?

### 1. 설치와 환경 설정

설치 자체는 매우 간단하다. Node.js가 설치되어 있다면 npm 명령어 하나로 끝난다.
```bash
npm install -g @anthropic-ai/claude-code
```

설치 후에는 API 키를 환경 변수로 등록해주어야 한다. Windows 환경이라면 아래 순서대로 진행하면 된다.

1. 윈도우 시작 버튼을 누르고 **`환경 변수`** 를 검색한 후, **[시스템 환경 변수 편집]** 을 클릭한다.
2. '시스템 속성' 창이 열리면 오른쪽 아래의 **[환경 변수(N)...]** 버튼을 클릭한다.
3. '사용자 변수' 목록에서 **[새로 만들기]** 를 클릭하고 Claude Code가 제공한 Path를 입력/저장한다.
4. 현재 열려있는 터미널을 종료하고 새로 열어야 변경 사항이 적용된다.

환경 변수 설정이 끝났다면, 프로젝트 루트 디렉토리에서 `claude` 명령어를 입력하면 바로 사용할 수 있다.
```bash
# 사용 예시
cd your-project
claude
```

### 2. dezeli.dev 디자인 수정에 활용하다

첫 번째 실전 투입은 dezeli.dev의 디자인 수정이었다. 오래전부터 마음에 걸렸던 UI 몇 군데를 손보기로 했다.

가장 먼저 느낀 점은 **파일을 수정하기 전에 먼저 물어봐준다**는 것이었다. 단순히 코드를 마구 고치는 게 아니라, 변경 사항을 먼저 제안하고 확인을 받은 뒤에 실행한다. 덕분에 엉뚱한 곳이 바뀌는 사고 없이 안심하고 작업을 맡길 수 있었다.

그리고 코드베이스를 파악하는 속도가 인상적이었다. 파일 구조와 컴포넌트 간의 관계를 빠르게 읽어내고, 수정 범위를 스스로 판단해서 작업해주었다. 결과적으로 디자인 수정은 꽤 만족스럽게 마무리됐다.

### 3. 가장 감동받은 기능: Git Commit 자동화

솔직히 말하면, 이것 하나 때문에 22달러가 납득이 됐다.

평소에 커밋 컨벤션을 나름대로 정해두긴 했지만, 막상 귀찮아서 대충 넘어가는 경우가 많았다. 그런데 Claude Code에 컨벤션 파일을 넘겨줬더니, 변경된 파일들을 분석해서 컨벤션에 맞게 커밋 메시지를 작성하고 파일별로 직접 커밋까지 쳐줬다.

```python
# 커밋 예시
Design : 네비게이션 바 반응형 레이아웃 개선

 - 모바일 환경 햄버거 메뉴 추가
 - 브레이크포인트별 레이아웃 조정

Design : 블로그 카드 컴포넌트 스타일 수정

 - hover 효과 및 그림자 추가
 - 썸네일 이미지 비율 통일
```

개발하면서 가장 귀찮은 작업 중 하나가 커밋이었는데, 그걸 대신 해주니 행복할 수밖에 없었다.

### 4. 총평: AI에 쓰는 돈은 투자일지도

한 달 22달러가 아깝지 않냐고 묻는다면, 지금 당장은 **아깝지 않다**고 답할 수 있다.

빠른 코드 파악, 신중한 파일 수정, 그리고 자동 커밋까지. 단순히 코드를 생성해주는 도구가 아니라, 개발 흐름 자체에 녹아드는 페어 프로그래머에 가깝다는 인상을 받았다.

앞으로 AI 도구에 들어가는 비용은 단순한 지출이 아니라, 나에 대한 투자로 바라봐야 할 것 같다. 적어도 Claude Code는 그 생각을 하게 만들기에 충분했다.
