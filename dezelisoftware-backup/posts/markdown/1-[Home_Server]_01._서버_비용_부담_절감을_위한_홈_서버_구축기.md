# [Home Server] 01. 서버 비용 부담 절감을 위한 홈 서버 구축기

개인 개발자에겐 항상 뒤따라오는 고민이 있다. 바로 어떻게하면 비용을 최소화할 것인가이다. 특히 24시간 가동해야 하는 서버 비용은 매달 고정적으로 지출되는 부담스러운 항목 중 하나이다.

### 1. AWS 프리티어의 한계와 새로운 대안
처음 선택했던 방법은 누구나 그렇듯 AWS EC2 프리티어였다. 초기 비용 절감에는 확실히 도움이 되었지만, 사양의 한계는 명확했다. t3.micro의 빈약한 성능과 12개월이라는 기간 제한은 프로젝트를 장기적으로 운영하고 꿈을 펼치기엔 제약 사항이 너무 많았다.
비용 고민이 깊어지던 와중, 방 한쪽 구석에 잠자고 있던 오디세이 게이밍 노트북이 떠올랐다. 한때는 화려한 그래픽의 게임을 돌리던 기기였지만, 이제는 구형이 되어 쓰임새를 잃어버린 노트북. 이 노트북을 서버로 활용한다면 추가 지출 없이 전기세만으로만 나만의 강력한 인프라를 구축할 수 있겠다는 확신이 들었다. 결심하는데는 오랜 시간이 걸리지 않았다. 이 과정자체가 개발자로서 엄청난 경험이 될 것이라 믿었기 때문이다.

### 2. 넘기힘든 벽: 아파트 네트워크
가장 먼저 미련 없이 노트북의 OS를 Ubuntu 22.04로 밀어버리고 서버 환경 설정을 시작했다. 하지만 곧바로 예상치 못한 난관에 부딪혔다. 바로 한국 아파트 특유의 폐쇄적인 네트워크 환경이었다.
외부에서 서버에 접속하려면 공유기 설정에서 포트포워딩(Port Forwarding)을 해주어야 한다. 하지만 공유기의 관리자 비밀번호를 몰랐기에 공유기를 초기화 해야 했는데 혼자 사는 집이 아니었기에 함부로 건들 수 없었다. 약간의 절망과 함께 다른 해결책을 찾아야만 했다.

### 3. Cloudflare Tunnel을 통한 해결
결국 내가 찾아낸 방법은 Cloudflare Tunnel이었다. 홈 서버에 다른 네트워크 기술을 활용한다는게 조금은 망설여졌지만 그럼에도 Cloduflare가 주는 장점들은 나를 설득하기에 충분했다.
- **포트포워딩 불필요:** 내부에서 외부로 터널을 뚫는 방식으로 공유기 설정을 건드릴 필요가없다.
- **보안 강화:** 공인 IP를 외부에 노출하지 않고도 도메인을 연결할 수 있다.
- **SSL 자동 적용:** 복잡한 인증서 설정 없이도 HTTPS 보안 접속을 기본으로 제공한다.
덕분에 보안과 접근성 문제를 한번에 해결할 수 있었다. 그렇게 나의 오디세이 노트북은 dezeli.dev를 지탱하는 든든한 인프라로 재탄생했다.

클라우드의 편리함을 포기하는 대신, 나만의 홈 서버라는 값진 결과와 경험을 얻었다. 다음 글에서는 실제로 Cloudflare Tunnel을 어떻게 설치하고 설정했는지 이야기해보려 한다.
